# 林月半子短视频封面 · 品牌基线与通用填空指引

## 全风格通用品牌基线（不管什么风格都不能动）

这些元素保证粉丝刷到封面能一眼认出"这是林月半子的视频"：

| 维度 | 固定值 |
|---|---|
| **画幅** | 3:4 竖版（1080×1440） |
| **人物** | 参考图 1（LQ 头像），30 多岁中国男性程序员，短发戴眼镜，穿深色休闲 T 恤，半身 |
| **表情默认值** | 自信微笑、看镜头、像导师分享技巧（绝不默认用震惊/瞪眼） |
| **标题原则** | 必须是人话，4-8 字主标题，技术行话必须翻译成大众词汇 |
| **视觉层级默认值** | 标题是第一主角，人物占画面 20-30%，不抢戏 |
| **水印** | 右下角 "@林月半子 🦞"，白字细描边 |
| **禁用** | 浅色背景失控、人物过大、标题行话、UI 抢主标题、过度发光特效 |

**关于参考图**：参考图只用于保持 LQ 的面部五官一致性，不用于锁定身体姿势、手势或衣服褶皱。各风格模板中的人物姿势和手势应根据构图重新设计。

## 通用填空指引

### {{人物表情}} 选项

默认使用"自信微笑看镜头"。如果风格需要，可从以下选择微调：

1. **自信微笑** — 默认，导师分享感
2. **专注认真** — 适合故障警示风、debug 内容
3. **轻松愉快** — 适合明亮 YouTuber 风、生活化内容
4. **思考状** — 适合观点、概念类
5. **交给模型决定** — 在默认基础上让模型根据主题微调

**禁止默认使用**：震惊瞪眼、夸张张嘴、戏剧化表情。

### {{人物手势}} 选项

手势**不必与参考图完全一致**，可以根据风格、构图和叙事需求让模型自然调整。参考图只用于保持面部五官一致性，不是用来锁定身体姿势、手势或肩膀角度的。

也就是说：即使参考图里是“手指抵下巴/指向胸前”的姿势，生成时也应该改用手指向 UI、指向标题、搭在桌面，或做其他符合当前构图的动作，而不是把同一个手部动作原样复制过来。

常用手势（按场景选）：

1. **食指轻抬指向重点** — 深色科技标题风默认，强调标题
2. **手指向 UI/产品界面** — 产品主视觉风、明亮 YouTuber 风、故障警示风
3. **双手分指左右对比** — 双屏对比叙事风
4. **手自然搭在桌面或轻触屏幕** — 工作台场景风
5. **单手轻触下巴/思考状** — 极简留白风、观点类内容
6. **交给模型决定** — 让模型根据构图自然选择，不强制指定

**注意**：
- 不要出现“双臂抱胸”“双手插兜”等封闭姿态（显得疏离）。
- 手势不能遮挡主标题或人脸。
- 如果某次生成的手势和参考图过于雷同，可以在 prompt 里加一句 `do NOT copy the reference pose 1:1, adjust the hand gesture to fit this composition`。

### {{标题}} 三行结构

推荐把主标题拆成 2-3 行，每行 3-5 字：

- 第 1 行：品类/对象（如 "AI 漫剧"）
- 第 2 行：方法/抓手（如 "一张画布"）
- 第 3 行：结果/承诺（如 "全搞定！"）

### {{UI 元素叙事}}

不管什么风格，UI 元素都要讲一个"问题 → 解决"或"工具 → 结果"的小故事：

- **问题状态**：报错、未读 999+、失败提示
- **解决状态**：成功对勾、整齐结果、绿色通过
- 两者用箭头或位置关系连接

### {{背景色调}} 选项

根据风格选择：

- 深色科技风 / 故障警示风：深灰 / 墨黑 / 深蓝黑
- 明亮 YouTuber 风：暖米 / 浅灰 / 奶油 / 浅蓝灰
- 极简留白风：白 / 米白 / 浅灰
- 产品主视觉风：白底或浅灰，让产品清晰
- 工作台场景风：自然光下的真实环境色

### {{字体风格}} 选项

1. **超粗黑体** — 冲击力最强，科技/教程首选
2. **圆润粗体** — 亲和力，生活化内容
3. **无衬线现代体** — 干净，极简留白风首选
4. **交给模型决定** — 默认

### {{字体颜色效果}} 选项

1. **白字重描边** — 通用安全牌
2. **纯色大字** — 背景简单时用
3. **渐变色** — 根据背景色调搭配
4. **故障/描边效果** — 故障警示风

## 文字渲染与乱码控制(全风格通用,2026-07 全风格实测总结)

用同一主题批量实测 11 种风格后确认的规律,填模板时必须执行:

1. **每个中文文字块的描述后面,必须追加一句**:
   `Render every Chinese character EXACTLY as written, no duplicated or missing characters`
   实测加了这句后 11 张封面的主标题/副标题/标签 0 错字。
2. **可读文字只放短语,不放长句。** 模型能稳定渲染 ≤8 字的中文短语(标题、报错横幅、标签、卡片角标);超过一句话的 UI 正文、搜索结果、聊天记录必定乱码。
3. **UI 截图内的长文本主动要求模糊化**,把"乱码"变成有意的设计。在 UI 元素描述里写:
   `blur or fade any long body text into soft unreadable texture; only the key phrase "XXX" must be rendered clearly`
   其中 XXX 是该截图唯一需要可读的短语(如报错横幅"无法访问外部网络"、开关名"Allowed Egress Hosts")。
4. **给副标题预留空位。** 底部副标题容易压在解决截图/人物胸口上,在模板里写明
   `reserve a clear zone for the bottom subtitle; it must not overlap key UI content or the subject's face`。
5. **标题与人物重叠时,人物在前(文字后置)。** 人物占比较大的风格(明亮 YouTuber 风、卡片展示人设风、手持设备展示风)里,标题很容易压住头发/头顶。在标题描述里写明:
   `the headline sits BEHIND the subject in z-order — where they overlap, the subject's head/hair occludes the text edge (text-behind-subject effect); text must never cover the hair or face`
   这既避免文字啃头,又自带爆款封面的纵深感。
